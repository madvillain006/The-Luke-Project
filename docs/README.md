# Luke Docs Map

This folder keeps durable project documentation. Generated run outputs belong under `artifacts/` unless they are promoted into one of the canonical docs below.

## Canonical Current Docs

- `HOSTILE_AUDIT_REPORT.md` - current audit verdict, risks, commands, and proof artifact pointers.
- `CURRENT_STATUS.md` - short operating snapshot.
- `REVIEW_READINESS.md` - readiness scores and claim classifications.
- `LIVE_BLOCKERS.md` - live execution and live-arming blockers.
- `TRADINGVIEW_INDICATOR.md` - TradingView/Pine safety notes and rerun commands.
- `TRADINGVIEW_HARDMODE_RESEARCH.md` - hard-mode strategy and slippage research summary.

## Historical Or Domain Notes

Research reports, parser findings, old review packets, and workflow notes are retained as context, not as the current readiness verdict. If a new audit produces raw output, keep the raw report in `artifacts/` and update the canonical doc instead of adding another top-level audit file here.
