What Was Rejected and Why

  • Grafting MemPalace/Hermes runtime as a dependency: Foreign framework — would add untested runtime risk with no operator-visible benefit over the existing Luke memory/radar layer.
  • Auto-accepting reference ideas into memory: No idea from a reference repo should enter Luke's trusted memory without human review. The reference_idea Radar lane exists precisely to prevent silent auto-adoption.
  • llm-routing-policy.js wiring to call buildIntelligencePacket directly: Deferred — the routing policy currently controls which provider is chosen. Injecting the packet at routing time couples policy to context assembly. The current approach (patch buildMergedChatContext) achieves the same enrichment result without that coupling.
  • Dashboard clutter (new panels, new cards): Phase 3 was kept to the minimum: one status line update in the shell. No new UI panels added.
